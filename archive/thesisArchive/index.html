This directory contains a collection of documents I wrote containing ideas relevant to my Ph. D. thesis. Below is an index of the documents with brief text descriptions and with links to the PDF files.

Curran Kelleher
2/12/2011

Here is an index of the material in this directory:
  • NSF GRFP Proposed Research - This is the most concise expression of my intended research direction: I want to build an interactive visualization system for the Semantic Web (knowledge base visualization and visual manipulation) and Universal Data Cube. This document was part of my submission for the NSF GRFP Fellowship. The fellowship recipients will be announced in April 2011.
    • The following documents supplement this one:
      • Previous Research
      • Personal Statement
  • Open Indicators Data Commons - This is one of the most concise documents explaining parts of my research vision to do with the Universal Data Cube and Semantic Web in relation to Weave.
  • Liquid Globe - A project idea sketch involving the development of an interactive tool using pan and zoom to interactively navigate through data cubes. The first step is to build a tool with three views: a time line for the time dimension, a map for the space dimension, and a fractal visualization of an additional taxonomic dimension such as industry category. I'd like to pursue this project to completion eventually, as it would serve as a tremendous platform for exploration of interactions and visualizations over data cubes.
    • The following documents supplement this one:
      • Quadstream paper draft - a paper draft for a set of algorithms for polygon generalization. The system described in this draft could be used for the choropleth map component of LiquidGlobe.
  • UDC Paper Draft - This is the universal data cube paper draft I produced while in Konstanz. I have not submitted this anywhere and don't have time now to dedicate to finishing it, perhaps it could be released as a technical report. The concepts in this paper draft serve as a conceptual foundation for many other proposed projects.
    • The following documents supplement this paper draft:
      • UDC For Weave - A document oriented toward introducing the UDC concepts and integrating them with Weave and the Open Indicators Consortium (OIC) community
      • IVPR universal data cube concept - slides introducing the concepts
      • konstanz initial presentation data cubes - slides of my first Konstanz presentation
      • OIC Weave Admin and ontology - slides for the OIC meeting
        • Introduces UDC
        • Also showcases screenshots of the data admin console prototype I did for Weave (which is now a dead project which could potentially be revisited to create a data admin console for the UDC / LiquidGlobe)
  • Visual Data Web - This document is an idea for an overarching framing and context for my thesis.
  • Pedigree Paper (very rough draft) - A very rough draft of a paper I am currently working on to do with interactive pedigree (family history diagram) visualization on multi-touch devices. This project may serve as an opportunity to implement an interactive graphics framework (compatible with multi-touch displays) which could be used to build the Liquid Globe and Visual RDF projects.
  • Theory of Visualization - This document is a kernel of a research project in model-based visualization system design which I would like to pursue in the long term in parallel with other projects such as LiquidGlobe, pedigree visualization, and Semantic Web visualization. I plan to treat this project as a "model repository" for precisely defining and integrating other projects.
    • Vis System Ideas - this document includes some related ideas
  • General Selection - An idea for a project that Andrada Tatu and I wrote together when I was in Konstanz
  • JyVis Paper - An old paper on a system I wrote, JyVis, from which I plan to draw ideas and code for future projects
  • Application State Historian Architecture Draft - Architectural ideas for a plugin-driven system supporting session history and synchronous collaboration (i.e. "Google Docs for Visualizations"). I would like to pursue this project eventually. I implemented a prototype of this system during VisWeek 2010.
