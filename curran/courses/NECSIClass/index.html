Project Summary:

 

Our group of six people did a purely theoretical/mathematical project on cellular automata (CA).

 

Stephen Wolfram came up with four distinct classes of one dimensional CAs: class 1 - stable/fixed, class 2 - cyclic, class 3 - chaotic (acyclic), and class 4 - local complex structures (the edge of chaos!). For our project, we extended these classes to 2D CA.

 

A "complexity profile" of a system is a concept introduced in the class. It is a plot of complexity versus scale. In the case of cellular automata, the complexity profile of the system can theoretically be calculated, based on the image of the cell grid at any given time.

 

Our hypothesis is that based on the characteristics of a plot of the system's complexity profile over time (a 3D surface plot of scale, complexity, and time), one can determine which of Wolfram's four classes the CA belongs to.

We then imported these images into Matlab for analysis. Matlab has a built in function for calculating the entropy of an image. This is our measure for "complexity." By varying the neighborhood of pixels used by Matlab to calculate entropy, this is a variation of "scale." This is how we calculated our complexity profile (actually it is an "entropy profile," but it should convey the same information (we may be wrong about this)).

 

We calculated and plotted (using Matlab) the entropy profile of the image of the CA at each timestep over time for four CAs (and five trial runs each)

 

Conclusions:

 

It seems that the class of a 2-dimensional CA can  indeed be determined by the characteristics of it's scale vs. complexity vs. time surface plot.
